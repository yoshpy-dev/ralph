# Planning rules

- Plans should define objective, scope, non-goals, acceptance criteria, verify plan, test plan, risks, and evidence.
- **Verify plan** covers: static analysis checks, spec compliance criteria, documentation drift checks, and evidence to capture.
- **Test plan** covers: unit tests, integration tests, regression tests, edge cases, and evidence to capture.
- Plans should constrain deliverables and contracts without freezing low-level implementation too early.
- Keep progress checklists current while the task is in flight.
- Record meaningful deviations from the original plan instead of silently drifting.
