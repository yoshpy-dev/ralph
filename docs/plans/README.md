# Plans

## Directory layout

- `active/` for in-flight plans
- `archive/` for completed plans
- `templates/` for reusable plan structures

## Naming

Suggested format:
- `YYYY-MM-DD-short-slug.md`

Example:
- `2026-03-27-login-form.md`

## Guidance

- Keep only genuinely active plans in `active/`
- Archive completed work instead of silently deleting history
- Update progress checklists while work is happening
- Link review and verify reports from the plan when possible
